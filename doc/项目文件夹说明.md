# Earth Online 项目文件夹说明

本文用于说明当前 `D:\游戏\earth-online` 项目的目录结构、各文件夹用途，以及后续开发时应注意的文件归属。

## 项目总体状态

当前项目处于“设定文档 + 美术素材 + Godot 工程雏形”阶段。

- `main` 是 Godot 游戏项目本体。
- `main/doc` 存放游戏设计、世界观、绘图提示词和项目说明。
- `main/assets` 存放已经导入 Godot 的游戏素材。
- `tools` 存放项目辅助脚本。
- `agent-sprite-forge` 是用于生成 2D 精灵和地图素材的外部工具/技能包，不属于游戏运行时代码。

当前 `main` 目录本身是 Git 仓库，项目根目录 `D:\游戏\earth-online` 不是 Git 仓库。

## 顶层目录

```text
D:\游戏\earth-online
├─ main/
├─ tools/
├─ agent-sprite-forge/
├─ FRAMERONIN_LOCAL.md
└─ start-frameronin.ps1
```

### `main/`

Godot 游戏项目主目录。后续真正的游戏场景、脚本、资源、配置都应优先放在这里。

当前包含：

- Godot 项目配置：`project.godot`
- Godot 图标：`icon.svg`
- Godot 导入缓存：`.godot/`
- 游戏素材：`assets/`
- 项目文档：`doc/`
- Git 相关配置：`.gitignore`、`.gitattributes`、`.editorconfig`

### `tools/`

项目辅助工具目录，目前包含：

```text
tools/
└─ build_walker_city_tiled.py
```

`build_walker_city_tiled.py` 用于把主城 tileset 图集处理成 Tiled/Godot 可用的瓦片地图数据。它会生成 tileset、Tiled JSON/TMX、预览图、碰撞调试图和 manifest。

注意：脚本默认输出路径为 `main/assets/maps/walker_city`，而当前主城区域图位于 `main/assets/map/walker_city/regions`。如果后续继续使用该脚本，建议统一 `map` / `maps` 的目录命名，避免资产分散。

### `agent-sprite-forge/`

2D 游戏美术资产生成工具包，包含生成角色精灵、动画帧、地图、prop pack、Godot 可编辑地图等工作流。

主要内容：

```text
agent-sprite-forge/
├─ README*.md
├─ requirements.txt
├─ src/
└─ skills/
   ├─ generate2dmap/
   └─ generate2dsprite/
```

用途说明：

- `skills/generate2dmap`：地图生成工作流，支持 tilemap、layered map、Godot/Tiled 数据、碰撞和区域元数据。
- `skills/generate2dsprite`：角色、怪物、NPC、法术、投射物、攻击特效等精灵图生成和后处理。
- `src`：示例图片、演示图和参考素材。
- `requirements.txt`：运行本地后处理脚本需要的 Python 依赖。

该目录更像“项目配套工具”或“素材流水线”，不是 Earth Online 游戏本体。游戏运行时代码不应直接依赖这里的 README 或技能文件。

### `FRAMERONIN_LOCAL.md` 和 `start-frameronin.ps1`

这两个文件是 FrameRonin 本地工具的历史启动说明和启动脚本。脚本期望存在：

```text
D:\游戏\earth-online\FrameRonin-main
```

但当前项目根目录下没有看到 `FrameRonin-main` 文件夹。因此它们目前应视为“保留的本地工具入口/历史记录”，不要把它们当作当前游戏本体的一部分。

如果后续恢复 FrameRonin 工具，应确认 `FrameRonin-main` 是否重新放回项目根目录，并检查 `start-frameronin.ps1` 中的路径是否仍然正确。

### `project.godot`

Godot 项目配置文件。

当前配置要点：

- Godot 配置版本：`config_version=5`
- 项目名称：`main`
- Godot 特性标记：`4.7`、`Forward Plus`
- 窗口拉伸模式：`canvas_items`
- 窗口比例策略：`expand`
- 3D 物理引擎：`Jolt Physics`
- Windows 渲染驱动：`d3d12`

虽然当前项目主要是 2D 像素风 RPG，但 Godot 配置里已启用 Forward Plus 和 Jolt。后续如果项目确定为纯 2D，可以再评估是否需要调整渲染和物理设置。

### `.godot/`

Godot 编辑器生成的本地缓存目录，包含导入缓存、编辑器布局、脚本文档缓存、shader cache 等。

一般规则：

- 不手动编辑 `.godot/` 里的文件。
- 不把 `.godot/` 当作游戏源码。
- `.gitignore` 已经配置忽略 `.godot/`。

### `.import` 文件

项目中有很多 `*.png.import`、`*.svg.import` 文件。这些是 Godot 为资源生成的导入配置文件。

一般规则：

- 图片资源进入 Godot 后会自动生成 `.import` 文件。
- 如果图片是正式资源，可以保留对应 `.import`。
- 不要手动大规模删除 `.import`，除非明确要让 Godot 重新导入资源。

### `assets/character/wukong/`

- `generated_sequence_frames/`：基于参考图生成的动作序列帧。

### `assets/character/wukong/generated_sequence_frames/`

悟空动作帧输出目录。

当前动作：

- `idle/`：待机动作，4 帧，建议 5-6 FPS。
- `run/`：跑步动作，4 帧，建议 8-10 FPS。
- `attack/`：攻击动作，6 帧，建议 9-12 FPS。

每个动作目录通常包含：

- `animation.gif`：动作预览。
- `raw-sheet.png`：原始生成图。
- `raw-sheet-clean.png`：清理后的图。
- `sheet-transparent.png`：透明背景精灵表。
- 单帧 PNG：例如 `wukong-idle-1.png`。
- `prompt-used.txt`：生成提示词。
- `pipeline-meta.json`：后处理参数、帧切分、边缘检测、对齐方式等元数据。

Godot 使用建议：

- 如果使用 `AnimatedSprite2D`，优先使用单帧透明 PNG。
- 如果使用 `Sprite2D + AnimationPlayer`，可以使用 `sheet-transparent.png`。
- `idle` 和 `run` 为 2x2 精灵表。
- `attack` 为 3x2 精灵表。

## 文件维护约定

### 文档

- 设计类文档放入 `main/doc/`。
- 地图或美术提示词放入 `main/doc/地图绘制提示词/` 或类似专题目录。
- 文件名优先使用清晰中文名，便于策划阶段阅读。

### 游戏资源

- 正式游戏资源放入 `main/assets/`。
- 角色资源按角色名分目录。
- 地图资源按地图名分目录。
- 生成过程文件可以保留在对应资源目录下，但最终进入游戏时应明确哪些是运行时资源、哪些只是预览或元数据。

### 工具和外部流水线

- 项目自写脚本放入 `tools/`。
- 外部工具包放在独立目录，例如 `agent-sprite-forge/`。
- 不建议把外部工具包和游戏运行时代码混在一起。

### Godot 缓存和导入文件

- `.godot/` 是编辑器缓存，不手动维护。
- `.import` 是 Godot 资源导入配置，随资源一起保留。
- 如果资源移动位置，Godot 可能重新生成 `.import` 文件。

## 当前需要注意的问题

1. `main` 是 Git 仓库，但当前多数文件处于未跟踪状态。后续提交前应先确认哪些文件需要纳入版本控制。
2. 根目录保留了 FrameRonin 启动脚本，但当前没有 `FrameRonin-main` 文件夹，使用前需要恢复或修正路径。
3. `tools/build_walker_city_tiled.py` 默认输出到 `assets/maps`，而现有地图素材在 `assets/map`，建议统一命名。
4. 当前 Godot 项目还没有看到正式 `scenes/`、`scripts/`、`data/` 目录，说明游戏逻辑尚未开始系统搭建。
5. 行者主城已有 7 张区域图，若要完成主城拼接，还需要补齐外城墙/防御入口，或重新规划主城区域切分。
